# 技术栈说明 — 多平台整蛊下载引导页

## 项目概述

一个基于「自动下载原神」梗的整蛊工具。通过微信将链接发送给朋友，页面自动检测设备类型，触发对应平台的游戏下载，1.5 秒后跳转原神官网，制造"手机/电脑自己开始下原神了"的整蛊效果。

## 架构决策详解

### 1. 单文件架构（Single HTML File）

**选择**：整个应用只有一个 `index.html`，零外部依赖，零构建步骤。

**理由**：项目逻辑简单（设备检测 + 跳转控制），引入框架反而增加复杂度和加载时间。原生 HTML/CSS/JS 在移动端更低延迟、更快执行。

### 2. 纯 location.href 跳转方案

**选择**：所有跳转均使用 `window.location.href`，不使用 `window.open`。

**理由**：`window.open` 在浏览器弹窗拦截器下不可靠，尤其在 `setTimeout` 回调中（非用户手势上下文）会被直接拦截。`location.href` 不受弹窗拦截器限制，100% 可靠。

```javascript
// 核心模式：下载 → 1.5s → 官网
window.location.href = downloadLink;
setTimeout(function () {
  window.location.href = officialSite;
}, 1500);
```

### 3. 设备检测策略

**选择**：UserAgent 字符串解析 + `maxTouchPoints` 辅助检测。

- `MicroMessenger` → 微信环境（显示引导页）
- `Android` + `Mobile` → Android 手机
- `iPhone` / `iPad` + `maxTouchPoints > 1` → iOS 设备
- 其他 → 桌面端兜底

**细节**：iPadOS 13+ 的 Safari UA 伪装为 Mac 桌面，通过 `navigator.maxTouchPoints > 1` 辅助识别触控设备。

### 4. 微信环境处理

**选择**：检测到微信内置浏览器时，不执行跳转，改为显示 CSS 动画引导页。

**理由**：微信内置浏览器对下载链接和非白名单域名有限制，强行跳转大概率失败。引导用户点击右上角「在浏览器中打开」是唯一可靠的绕开微信限制的方式。

### 5. 双链接跳转时序

```
T+0.0s: location.href = 下载直链（触发浏览器下载）
T+1.5s: location.href = 原神官网（当前页跳转）
```

浏览器下载管理器在页面导航后仍会继续下载任务，两个操作互不干扰。1.5 秒延迟确保下载请求先发出。
